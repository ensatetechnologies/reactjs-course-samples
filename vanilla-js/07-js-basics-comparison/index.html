2Ô∏è‚É£ Functions

JavaScript functions.js 
// No parameter validation
function calculateTotal(price, quantity) {
  return price * quantity
}

// These cause bugs!
calculateTotal("10", 5)  // "1010101010" üò±
calculateTotal()          // NaN üò±

// Arrow function

  return `Hello, ${name}!`
}
VS
TypeScript functions.ts 
// Typed parameters and return
function calculateTotal(
  price: number, 
  quantity: number
): number {
  return price * quantity
}

// TypeScript catches these!
calculateTotal("10", 5)  // ‚ùå Error!
calculateTotal()          // ‚ùå Error!

// Typed arrow function
const greet = (name: string): string => {
  return `Hello, ${name}!`
}

3Ô∏è‚É£ Objects & Interfaces

JavaScript objects.js 
// Plain object
const user = {
  name: "John",
  age: 25,
  email: "john@example.com"
}

// No structure enforcement
user.nam = "Jane"  // Typo creates new prop! üò±
user.age = "old"   // Wrong type, no error üò±

// Missing property?
console.log(user.address)  // undefined
VS
TypeScript objects.ts 
// Interface defines structure
interface User {
  name: string
  age: number
  email: string
  address?: string  // Optional
}

const user: User = {
  name: "John",
  age: 25,
  email: "john@example.com"
}

user.nam = "Jane"  // ‚ùå Error: 'nam' doesn't exist
user.age = "old"   // ‚ùå Error: Type mismatch

4Ô∏è‚É£ Arrays

JavaScript arrays.js 
// Arrays can mix types
const numbers = [1, 2, 3]
numbers.push("four")  // No error! üò±

// Mixed array allowed
const mixed = [1, "two", true]

// No autocomplete for items
const users = [
  { name: "John", age: 25 },
  { name: "Jane", age: 30 }
]
users[0].nam  // undefined (typo)
VS
TypeScript arrays.ts 
// Typed arrays
const numbers: number[] = [1, 2, 3]
numbers.push("four")  // ‚ùå Error!

// Union type for mixed
const mixed: (number | string)[] = [1, "two"]

// Full autocomplete!
interface User { name: string; age: number }

const users: User[] = [
  { name: "John", age: 25 },
  { name: "Jane", age: 30 }
]
users[0].nam  // ‚ùå Error!

üìä Key Takeaways

üü® JavaScript

  • Dynamic typing (runtime)
  • Flexible, less verbose
  • Errors found when code runs
  • Faster prototyping
  • Lower learning curve

üü¶ TypeScript

  • Static typing (compile-time)
  • Structured, explicit
  • Errors caught before running
  • Better IDE support
  • Self-documenting code
