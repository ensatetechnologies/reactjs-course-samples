What is Conditional Rendering?

Conditional rendering means showing different UI elements based on certain conditions. Just like JavaScript's if statements, React lets you render different things depending on the state.

üéØ Real-World Examples

  • Login/Logout button - Show "Login" if not authenticated, "Logout" if authenticated
  • Loading spinner - Show spinner while fetching data, content when loaded
  • Error messages - Show only when there's an error
  • Empty states - Show "No items" when list is empty
  • Admin features - Show only for users with admin role

üí° Key Concept
In React, you can't use if statements directly inside JSX (the return statement). You need to use expressions like ternary operators or the && operator, or put the if statement before the return.

Using if/else for Conditional Rendering

The most straightforward approach: use regular JavaScript if/else statements before the return statement.

üìù Basic if/else Pattern

  if/else Before Return
function Greeting({ isLoggedIn, userName }) {
  // if/else BEFORE the return statement
  if (isLoggedIn) {
    return (
      <div>
        <h1>Welcome back, {userName}!</h1>
        <button>Logout</button>
      </div>
    );
  } else {
    return (
      <div>
        <h1>Please log in</h1>
        <button>Login</button>
      </div>
    );
  }
}

// Usage
<Greeting isLoggedIn={true} userName="John" />
// ‚Üí Renders: "Welcome back, John!" with Logout button

<Greeting isLoggedIn={false} />
// ‚Üí Renders: "Please log in" with Login button

    <div className="profile">
      <h1>{student.name}</h1>
      <p>Age: {student.age}</p>
      <p>Grade: {student.grade}</p>
    </div>

}

Inline Conditional with Ternary

The ternary operator is the most common way to conditionally render in JSX. It works inline and returns one of two values based on a condition.

Syntax

condition ? ifTrue : ifFalse

üìù Basic Ternary Examples

  Ternary in JSX

      </h1>
      
      {/* Element ternary */}
      {isLoggedIn ? (
        <button>Logout</button>
      ) : (
        <button>Login</button>
      )}
    </div>
  );
}

Show Something OR Nothing

The && operator is perfect when you want to show something only if a condition is true, and nothing otherwise. No "else" needed!

How It Works

condition && element
‚Ä¢ If condition is truthy ‚Üí returns element
‚Ä¢ If condition is falsy ‚Üí returns false (renders nothing)

üìù Basic && Examples

  && Operator Usage

‚ö†Ô∏è Gotcha: The Zero Problem

üö® Watch Out for Numbers!
If the left side is 0, React will render 0 instead of nothing!
  The Zero Gotcha

  return (
    <div>
      {/* ‚ùå WRONG - Will render "0" when count is 0 */}

      {/* ‚úÖ ALSO CORRECT - Convert to boolean */}
      {!!count && <span>{count} messages</span>}
      
      {/* ‚úÖ OR use ternary for "else" case */}
      {count > 0 ? <span>{count} messages</span> : <span>No messages</span>}

Handling Multiple Conditions

When you have many possible values (like status codes or types), switch statements or object mapping are cleaner than many if/else blocks.

üî¢ Switch Statement

  Switch Pattern
function StatusIcon({ status }) {
  // Switch BEFORE return
  const getStatusContent = () => {
    switch (status) {
      case 'loading':
        return <span className="loading">‚è≥ Loading...</span>;
      
      case 'success':
        return <span className="success">‚úÖ Success!</span>;
      
      case 'error':
        return <span className="error">‚ùå Error!</span>;
      
      case 'warning':
        return <span className="warning">‚ö†Ô∏è Warning</span>;
      
      default:
        return <span className="unknown">‚ùì Unknown</span>;
    }
  };

  return <div className="status">{getStatusContent()}</div>;
}

Understanding Keys

Why Keys Matter

Keys help React identify which items have changed, been added, or removed. They are essential for efficient list rendering and preventing bugs.

üîë What is a Key?

üìã
Array
[A, B, C]
‚Üí
üîë
Keys
Unique identifiers
‚Üí
‚öõÔ∏è
React
Tracks changes

‚úÖ Good Key Choices

  Good Keys
// ‚úÖ BEST: Database ID (guaranteed unique)
{students.map(student => (
  <StudentCard key={student.id} student={student} />
))}

// ‚úÖ GOOD: UUID or unique identifier
{items.map(item => (
  <Item key={item.uuid} data={item} />
))}

// ‚úÖ GOOD: Unique string value (if guaranteed unique)
{countries.map(country => (
  <option key={country.code} value={country.code}>
    {country.name}
  </option>
))}

// ‚úÖ OK: Index (only for static lists that never reorder)
{staticMenuItems.map((item, index) => (
  <MenuItem key={index} item={item} />
))}

‚ùå Bad Key Choices

  Bad Keys
// ‚ùå BAD: Random number (changes every render!)
{items.map(item => (
  <Item key={Math.random()} data={item} />  // NEVER do this!
))}

// ‚ùå BAD: Index for dynamic lists (will cause bugs)
{todoList.map((todo, index) => (
  <TodoItem key={index} todo={todo} />  // Buggy if list changes!
))}

// ‚ùå BAD: Non-unique values
{students.map(student => (
  <StudentCard key={student.grade} student={student} />  // Multiple A's!
))}

üêõ What Happens Without Good Keys?

üö® Index Key Bug Example
When using index as key for a dynamic list that can be reordered, React gets confused about which item is which.
  The Index Key Bug
// Initial list: [A, B, C] with indexes [0, 1, 2]
// User deletes B
// New list: [A, C] with indexes [0, 1]

// ‚ùå With index keys:
// React thinks: "key 0 is A (same), key 1 is now C (was B)"
// React updates B's content to C instead of removing B!
// This causes input state, animations, etc. to be wrong

// ‚úÖ With unique ID keys:
// React knows: "id-A still exists, id-B is gone, id-C still exists"
// React correctly removes B's element

üîë Key Rules Summary

Rule Explanation
Must be unique Among siblings (not globally)
Must be stable Same item = same key across renders
Use IDs when possible Database IDs, UUIDs are best
Index only for static lists Never add, remove, or reorder
Don't generate in render No Math.random() or Date.now()

Transforming Lists Before Rendering

Often you need to filter, sort, or transform data before displaying it. Chain array methods before map() to achieve this.

üîç Filtering Lists

  Filter Examples
function FilteredStudentList({ students }) {
  const [gradeFilter, setGradeFilter] = useState('all');
  const [searchTerm, setSearchTerm] = useState('');

  // Filter the array based on state
  const filteredStudents = students
    // Filter by grade
    .filter(student => {
      if (gradeFilter === 'all') return true;
      return student.grade === gradeFilter;
    })
    // Filter by search term
    .filter(student => {
      if (!searchTerm) return true;
      return student.name.toLowerCase().includes(searchTerm.toLowerCase());
    });

  return (
    <div>
      {/* Filter controls */}
      <input
        type="text"
        placeholder="Search by name..."
        value={searchTerm}
        onChange={(e) => setSearchTerm(e.target.value)}
      />
      
      <select value={gradeFilter} onChange={(e) => setGradeFilter(e.target.value)}>
        <option value="all">All Grades</option>
        <option value="A">Grade A</option>
        <option value="B">Grade B</option>
        <option value="C">Grade C</option>
      </select>
      
      {/* Show count */}
      <p>Showing {filteredStudents.length} of {students.length} students</p>
      
      {/* Render filtered list */}
      <ul>
        {filteredStudents.map(student => (
          <li key={student.id}>{student.name} - {student.grade}</li>
        ))}
      </ul>
      
      {/* Empty state */}
      {filteredStudents.length === 0 && (
        <p>No students match your filters</p>
      )}
    </div>
  );
}

function SortableStudentList({ students }) {
  const [sortBy, setSortBy] = useState('name');
  const [sortOrder, setSortOrder] = useState('asc');

  // Sort the array
  const sortedStudents = [...students].sort((a, b) => {
    let comparison = 0;
    
    switch (sortBy) {
      case 'name':
        comparison = a.name.localeCompare(b.name);
        break;
      case 'age':
        comparison = a.age - b.age;
        break;
      case 'grade':
        comparison = a.grade.localeCompare(b.grade);
        break;
      default:
        comparison = 0;
    }
    
    // Reverse for descending order
    return sortOrder === 'asc' ? comparison : -comparison;
  });

  const toggleSort = (field) => {
    if (sortBy === field) {
      setSortOrder(sortOrder === 'asc' ? 'desc' : 'asc');
    } else {
      setSortBy(field);
      setSortOrder('asc');
    }
  };

  return (
    <table>
      <thead>
        <tr>
          <th onClick={() => toggleSort('name')}>
            Name {sortBy === 'name' && (sortOrder === 'asc' ? '‚Üë' : '‚Üì')}
          </th>
          <th onClick={() => toggleSort('age')}>
            Age {sortBy === 'age' && (sortOrder === 'asc' ? '‚Üë' : '‚Üì')}
          </th>
          <th onClick={() => toggleSort('grade')}>
            Grade {sortBy === 'grade' && (sortOrder === 'asc' ? '‚Üë' : '‚Üì')}
          </th>
        </tr>
      </thead>
      <tbody>
        {sortedStudents.map(student => (
          <tr key={student.id}>
            <td>{student.name}</td>
            <td>{student.age}</td>
            <td>{student.grade}</td>
          </tr>
        ))}
      </tbody>
    </table>
  );
}
‚ö†Ô∏è Don't Mutate Original Array!
sort() mutates the original array! Always create a copy first with [...array] or array.slice().

  const [filter, setFilter] = useState('all');
  const [searchTerm, setSearchTerm] = useState('');
  const [isLoading, setIsLoading] = useState(false);

  // Filter and search
  const filteredStudents = students
    .filter(s => filter === 'all' || s.grade === filter)
    .filter(s => s.name.toLowerCase().includes(searchTerm.toLowerCase()));

  // Handlers
  const deleteStudent = (id) => {
    setStudents(prev => prev.filter(s => s.id !== id));
    if (selectedId === id) setSelectedId(null);
  };

  // Loading state
  if (isLoading) {
    return <div className="loading">Loading students...</div>;
  }

  return (
    <div className="dashboard">
      <h1>Student Dashboard</h1>
      
      {/* Stats */}
      <div className="stats">
        <div>Total: {students.length}</div>
        <div>Honor Roll: {students.filter(s => s.isHonorRoll).length}</div>
        <div>Low Attendance: {students.filter(s => s.attendance < 80).length}</div>
      </div>
      
      {/* Controls */}
      <div className="controls">
        <input
          type="text"
          placeholder="Search students..."
          value={searchTerm}
          onChange={(e) => setSearchTerm(e.target.value)}
        />
        
        <select value={filter} onChange={(e) => setFilter(e.target.value)}>
          <option value="all">All Grades</option>
          {Object.keys(GRADE_CONFIG).map(grade => (
            <option key={grade} value={grade}>Grade {grade}</option>
          ))}
        </select>
      </div>
      
      {/* Results count */}
      <p>
        Showing {filteredStudents.length} of {students.length} students
        {searchTerm && ` matching "${searchTerm}"`}
      </p>
      
      {/* Student grid or empty state */}
      {filteredStudents.length > 0 ? (
        <div className="grid">
          {filteredStudents.map(student => (
            <StudentCard
              key={student.id}
              student={student}
              isSelected={student.id === selectedId}
              onSelect={setSelectedId}
              onDelete={deleteStudent}
            />
          ))}
        </div>
      ) : (
        <div className="empty-state">
          <p>No students found</p>
          {(searchTerm || filter !== 'all') && (
            <button onClick={() => { setSearchTerm(''); setFilter('all'); }}>
              Clear Filters
            </button>
          )}
        </div>
      )}
    </div>
  );
}

export default StudentDashboard;
