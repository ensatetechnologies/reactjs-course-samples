What is React?

React.js is one of the most popular frontend JavaScript libraries. The creators call it "The library for web and native user interfaces."

üí° What is a Library?

A library is a collection of functionalities that you can use in your code to achieve results that would normally require more code. React provides utility functionalities that help you create interactive and reactive UIs.

üö´ The Problem with Vanilla JavaScript

Vanilla JavaScript refers to JavaScript without any frameworks or libraries. While it results in smaller download sizes, it has significant downsides for complex UIs.

‚ùå Imperative Approach

Step-by-step DOM instructions

  vanilla.js

const paragraphEl = document.querySelector('p');

  paragraphEl.textContent = 'Text was changed!';
}

buttonEl.addEventListener('click', updateTextHandler);

~75% of code is DOM manipulation!

VS

‚úÖ Declarative Approach

Define desired states

  App.jsx

  return (
    <>
      <button onClick={() => setText('Changed!')}>
        Click to change text
      </button>
      <p>{text}</p>
    </>
  );
}

üåê Single Page Applications (SPAs)

React can handle URL path changes and update the page accordingly. A website using React for all subpages is called a SPA because it contains only one HTML file (typically index.html).

Props (properties) make components configurable and reusable. They work just like HTML attributes but for custom components.

‚ö†Ô∏è Props are Read-Only
Props are immutable. A component cannot modify its own props. When a component needs to change its props, it must "ask" its parent component to pass different props.

State: Making UIs Interactive

Regular variables don't trigger re-renders. React's useState hook lets you add state to components.

üé£ The useState Hook

  useState Syntax
import { useState } from 'react';

function Counter() {
  // Destructure the array: [currentValue, setterFunction]
  const [count, setCount] = useState(0);

  return (
    <div>
      <p>Count: {count}</p>
      <button onClick={() => setCount(count + 1)}>
        Increment
      </button>
    </div>
  );
}

üí° How useState Works

  1. Call setValue(newValue) to update state
  2. React re-executes the component function
  3. useState returns a new array with the updated value
  4. React updates the DOM where needed

üìã State Can Hold Any Value

Primitives

useState(0)
useState('')
useState(true)

Arrays

useState([])
useState([1, 2, 3])

Objects

useState({})
useState({name: ''})

üìù Recap: Events & State

  • Event handlers are functions passed to event props like onClick
  • Pass function reference, not the function call: onClick={handleClick}
  • Use useState to add reactive state to components
  • State changes trigger component re-renders
  • Each component instance has its own isolated state

When multiple components need to share state, move it to their closest common parent. This is called "lifting state up".

üéπ Example: Accordion Component

  Accordion.jsx - Lifting State Up
function Accordion() {
  // State lives in parent - "single source of truth"
  const [activeIndex, setActiveIndex] = useState(0);

  return (
    <>
      <Panel
        title="About"
        isActive={activeIndex === 0onShow={() => setActiveIndex(0)}
      >
        About content here...
      </Panel>
      <Panel
        title="FAQ"
        isActive={activeIndex === 1}
        onShow={() => setActiveIndex(1)}
      >
        FAQ content here...
      </Panel>
    </>
  );
}

üîë Preserving and Resetting State

‚úÖ State Preserved

Same component at same position in the tree keeps its state.

üîÑ State Reset

Different component at same position, or same component with different key.

üí° Force Reset with Key
Use the key prop to tell React to treat a component as different: <Form key={selectedId} />

üìù State Management Principles

  • State is tied to position in the render tree
  • Lift state to the closest common parent when sharing
  • Use key prop to reset component state
  • Each piece of state should have a single "source of truth"

Chapter 6: useReducer Hook
‚ñº

What is useReducer?

useReducer is an alternative to useState for complex state logic. It consolidates state updates in a single reducer function.

üìä useState vs useReducer

useState

Simple state, few updates

const [count, setCount] = useState(0);
setCount(count + 1);
VS

useReducer

Complex state, many updates

const [state, dispatch] = useReducer(
  reducer, initialState
);
dispatch({ type: 'increment' });

üîß useReducer Syntax

  TasksReducer.jsx
import { useReducer } from 'react';

// Reducer function - handles all state updates
function tasksReducer(state, action) {
  switch (action.type) {
    case 'added':
      return [...state, {
        id: action.id,
        text: action.text,
        done: false
      }];
    case 'changed':
      return state.map(t => 
        t.id === action.task.id ? action.task : t
      );
    case 'deleted':
      return state.filter(t => t.id !== action.id);
    default:
      throw Error('Unknown action: ' + action.type);
  }
}

function TaskApp() {
  const [tasks, dispatch] = useReducer(tasksReducer, []);

  function handleAddTask(text) {
    dispatch({
      type: 'added',
      id: nextId++,
      text: text
    });
  }

  function handleDeleteTask(taskId) {
    dispatch({
      type: 'deleted',
      id: taskId
    });
  }

  // ... render tasks
}

‚úÖ When to Use useReducer

Use useReducer when:

  • Many event handlers modify state similarly
  • State updates are complex
  • You want to separate "what happened" from "how to update"
  • You need easier testing

Stick with useState when:

  • State is simple (primitives)
  • Few state updates needed
  • Updates are straightforward

The Problem: Prop Drilling

Passing props through many intermediate components that don't use them is called "prop drilling". Context lets you "teleport" data to components that need it.

Parent (has data)
‚Üì props ‚Üì props ‚Üì props
Intermediate Component
‚Üì props ‚Üì props ‚Üì props
Another Component
‚Üì props ‚Üì props ‚Üì props
Child (needs data)

‚ú® Context: Three Steps

  1. Create the context with createContext()
  2. Provide the context value from a parent component
  3. Consume the context with useContext()

üîß Context Implementation

  ThemeContext.jsx
import { createContext, useContext, useState } from 'react';

// Step 1: Create context
const ThemeContext = createContext('light');

// Step 2: Provider component
function ThemeProvider({ children }) {
  const [theme, setTheme] = useState('light');
  
  return (
    <ThemeContext value={{ theme, setTheme }}>
      {children}
    </ThemeContext>
  );
}

// Step 3: Consume with useContext
function ThemedButton() {
  const { theme, setTheme } = useContext(ThemeContext);
  
  return (
    <button
      onClick={() => setTheme(theme === 'light' ? 'dark' : 'light')}
      className={theme}
    >
      Toggle Theme
    </button>
  );
}

üéØ Common Use Cases for Context

üé® Theming

Dark mode, color schemes

üë§ Current User

Authentication state

üß≠ Routing

Current route information

üìä Global State

App-wide data management

‚ö†Ô∏è Before Using Context
1. Try passing props first - it makes data flow explicit.
2. Extract components - pass JSX as children instead of drilling data.

üìù Context API Recap

  • Context lets a parent provide data to the entire tree below it
  • Create with createContext(), provide with <Context value={...}>
  • Consume with useContext(MyContext)
  • Context passes through any components in the middle
  • Different contexts don't override each other
