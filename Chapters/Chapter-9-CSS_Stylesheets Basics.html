üéØ CSS Selectors

1.2 The Box Model - Understanding Element Sizing
HTML/CSS

Every Element is a Box

The CSS Box Model describes how every HTML element is rendered as a rectangular box with content, padding, border, and margin.

Box Model Visualization
MARGIN
BORDER
PADDING
CONTENT
  box-model.css
.box {
  /* CONTENT - the actual content area */
  width: 300px;
  height: 200px;

  /* PADDING - space inside the border */
  padding: 20px;              /* all sides */
  padding: 10px 20px;         /* vertical | horizontal */
  padding: 10px 20px 15px 25px; /* top | right | bottom | left */
  padding-top: 10px;          /* individual sides */

  /* BORDER - the edge of the box */
  border: 2px solid #333;     /* width | style | color */
  border-radius: 12px;        /* rounded corners */
  border-top: 3px dashed red; /* individual sides */

  /* MARGIN - space outside the border */
  margin: 20px;               /* same syntax as padding */
  margin: 0 auto;            /* center horizontally! */
}

üìê Box-Sizing: The Game Changer

‚ùå content-box (Default)

Width/height = content only. Padding & border are ADDED. 

.box {
  width: 200px;
  padding: 20px;
  border: 10px solid;
}
/* Actual width: 200 + 40 + 20 = 260px! */

‚úÖ border-box (Recommended)

Width/height = total box. Padding & border are INCLUDED. 

.box {
  box-sizing: border-box;
  width: 200px;
  padding: 20px;
  border: 10px solid;
}
/* Actual width: 200px (as expected!) */
üí° Always Use border-box
Add this to every project: *, *::before, *::after { box-sizing: border-box; }

1.3 Flexbox - One-Dimensional Layouts
HTML/CSS
‚ñº

Flexbox: The Layout Revolution

Flexbox makes it easy to align and distribute space among items in a container, even when their size is unknown or dynamic.

üéØ Enable Flexbox

  Enable Flexbox
.container {
  display: flex;  /* This is all you need to start! */
}

üì¶ Container Properties (Parent)

  flexbox-container.css
.flex-container {
  display: flex;

  /* DIRECTION - main axis */
  flex-direction: row;            /* default: left to right */
  flex-direction: row-reverse;    /* right to left */
  flex-direction: column;         /* top to bottom */
  flex-direction: column-reverse; /* bottom to top */

  /* WRAPPING */
  flex-wrap: nowrap;   /* default: squeeze everything */
  flex-wrap: wrap;     /* wrap to next line */

  /* JUSTIFY-CONTENT - main axis alignment */
  justify-content: flex-start;    /* pack at start */
  justify-content: flex-end;      /* pack at end */
  justify-content: center;        /* center items */
  justify-content: space-between; /* equal space between */
  justify-content: space-around;  /* equal space around */
  justify-content: space-evenly;  /* truly equal spacing */

  /* ALIGN-ITEMS - cross axis alignment */
  align-items: stretch;    /* default: fill height */
  align-items: flex-start; /* align to top */
  align-items: flex-end;   /* align to bottom */
  align-items: center;     /* center vertically */

  /* GAP - space between items */
  gap: 20px;      /* both row and column gap */
  gap: 10px 20px; /* row gap | column gap */
}

üß© Item Properties (Children)

  flexbox-items.css
.flex-item {
  /* FLEX GROW - how much to grow */
  flex-grow: 0;   /* default: don't grow */
  flex-grow: 1;   /* grow to fill space */

  /* FLEX SHRINK - how much to shrink */
  flex-shrink: 1; /* default: can shrink */
  flex-shrink: 0; /* don't shrink */

  /* FLEX BASIS - initial size */
  flex-basis: auto;  /* default: use width/height */
  flex-basis: 200px; /* start at 200px */

  /* SHORTHAND - grow | shrink | basis */
  flex: 1;         /* flex: 1 1 0% - grow equally */
  flex: 0 0 200px; /* fixed 200px, no grow/shrink */

  /* ALIGN-SELF - override parent's align-items */
  align-self: flex-end; /* align this item only */

  /* ORDER - change visual order */
  order: -1; /* move to front */
  order: 1;  /* move to back */
}

1.4 CSS Grid - Two-Dimensional Layouts
HTML/CSS
‚ñº

CSS Grid: Complete Layout Control

CSS Grid is a two-dimensional layout system for creating complex layouts with rows AND columns. It's perfect for page layouts and complex component structures.

üìê Grid Container Properties

  grid-container.css
.grid-container {
  display: grid;

  /* DEFINE COLUMNS */
  grid-template-columns: 200px 200px 200px;     /* 3 fixed columns */
  grid-template-columns: 1fr 1fr 1fr;          /* 3 equal columns */
  grid-template-columns: repeat(3, 1fr);       /* same as above */
  grid-template-columns: 1fr 2fr 1fr;          /* middle is 2x */
  grid-template-columns: 200px 1fr 200px;      /* sidebar layout */
  grid-template-columns: repeat(auto-fit, minmax(250px, 1fr)); /* responsive! */

  /* DEFINE ROWS */
  grid-template-rows: 100px auto 100px;  /* header, main, footer */
  grid-auto-rows: minmax(100px, auto); /* min 100px, grow as needed */

  /* GAP */
  gap: 20px;          /* both row and column */
  row-gap: 10px;      /* row gap only */
  column-gap: 20px;   /* column gap only */

  /* ALIGNMENT */
  justify-items: center;  /* horizontal align items */
  align-items: center;    /* vertical align items */
  place-items: center;    /* shorthand for both */
}

üß© Grid Item Properties

  grid-items.css
.grid-item {
  /* SPAN COLUMNS */
  grid-column: 1 / 3;      /* start at 1, end at 3 (spans 2) */
  grid-column: span 2;     /* span 2 columns from current */
  grid-column: 1 / -1;     /* full width (first to last) */

  /* SPAN ROWS */
  grid-row: 1 / 3;         /* start at row 1, end at row 3 */
  grid-row: span 2;        /* span 2 rows */

  /* SHORTHAND */
  grid-area: 1 / 1 / 3 / 3; /* row-start / col-start / row-end / col-end */
}

üèóÔ∏è Grid Template Areas (Powerful!)

  grid-template-areas.css
.page-layout {
  display: grid;
  grid-template-columns: 200px 1fr 200px;
  grid-template-rows: auto 1fr auto;
  grid-template-areas:
    "header  header  header"
    "sidebar main    aside"
    "footer  footer  footer";
  min-height: 100vh;
  gap: 20px;
}

.header  { grid-area: header;  }
.sidebar { grid-area: sidebar; }
.main    { grid-area: main;    }
.aside   { grid-area: aside;   }
.footer  { grid-area: footer;  }

‚öñÔ∏è Flexbox vs Grid - When to Use Which?

Use Flexbox When... Use Grid When...
One-dimensional layout (row OR column) Two-dimensional layout (rows AND columns)
Content should dictate layout Layout should dictate content
Navigation bars, button groups Page layouts, dashboards
Centering single items Complex card grids
üí° Pro Tip: Combine Them!
Use Grid for overall page layout, Flexbox for component internals. They work great together!

1.5 Responsive Design - Mobile-First Approach
HTML/CSS
‚ñº

Making Websites Work on All Devices

Responsive Design ensures your website looks great on phones, tablets, and desktops. The Mobile-First approach means designing for small screens first, then adding complexity.

üìê The Viewport Meta Tag (Required!)

  index.html
<head>
  <!-- This tells mobile browsers to use actual device width -->
  <meta name="viewport" content="width=device-width, initial-scale=1.0">
</head>

üìè Media Queries

  responsive.css
/* ========== MOBILE FIRST APPROACH ========== */

/* BASE STYLES - Mobile (default) */
.container {
  padding: 1rem;
}

.card-grid {
  display: grid;
  grid-template-columns: 1fr;  /* 1 column on mobile */
  gap: 1rem;
}

.nav-links {
  display: none;  /* Hidden on mobile (show hamburger instead) */
}

/* TABLET - 768px and up */
@media (min-width: 768px) {
  .container {
    padding: 2rem;
  }

  .card-grid {
    grid-template-columns: repeat(2, 1fr);  /* 2 columns */
  }

  .nav-links {
    display: flex;  /* Show nav on tablet+ */
  }

  .hamburger {
    display: none;  /* Hide hamburger on tablet+ */
  }
}

/* DESKTOP - 1024px and up */
@media (min-width: 1024px) {
  .container {
    max-width: 1200px;
    margin: 0 auto;
  }

  .card-grid {
    grid-template-columns: repeat(3, 1fr);  /* 3 columns */
    gap: 2rem;
  }
}

/* LARGE DESKTOP - 1440px and up */
@media (min-width: 1440px) {
  .card-grid {
    grid-template-columns: repeat(4, 1fr);  /* 4 columns */
  }
}

üìê Responsive Units

Relative Units

  • % - Relative to parent
  • em - Relative to font-size
  • rem - Relative to root font-size
  • vw/vh - Viewport width/height

Fixed Units

  • px - Pixels (avoid for layout)
  • pt - Points (print only)

Best Practice

  • Use rem for typography
  • Use % or fr for widths
  • Use vh/vw for full-screen

Transitions animate property changes. Animations create complex multi-step movements with keyframes.

üîÑ Transitions (Simple)

  transitions.css
.button {
  background: #22d3ee;
  transform: scale(1);
  
  /* Transition: property | duration | timing | delay */
  transition: all 0.3s ease;
  
  /* Or specify individual properties */
  transition: background 0.3s ease, transform 0.2s ease;
}

.button:hover {
  background: #06b6d4;
  transform: scale(1.05) translateY(-2px);
}

/* Timing functions */
.ease     { transition-timing-function: ease; }      /* default */
.linear   { transition-timing-function: linear; }
.ease-in  { transition-timing-function: ease-in; }  /* slow start */
.ease-out { transition-timing-function: ease-out; } /* slow end */
.bounce   { transition-timing-function: cubic-bezier(0.68, -0.55, 0.265, 1.55); }

üé¨ Keyframe Animations (Complex)

  animations.css
/* Define the animation */
@keyframes fadeIn {
  from {
    opacity: 0;
    transform: translateY(20px);
  }
  to {
    opacity: 1;
    transform: translateY(0);
  }
}

@keyframes pulse {
  0%, 100% {
    transform: scale(1);
  }
  50% {
    transform: scale(1.1);
  }
}

@keyframes spin {
  from { transform: rotate(0deg); }
  to   { transform: rotate(360deg); }
}

/* Apply the animation */
.fade-in {
  animation: fadeIn 0.5s ease forwards;
}

.pulsing {
  animation: pulse 2s ease-in-out infinite;
}

.spinner {
  animation: spin 1s linear infinite;
}

/* Animation properties breakdown */
.animated {
  animation-name: fadeIn;
  animation-duration: 0.5s;
  animation-timing-function: ease;
  animation-delay: 0s;
  animation-iteration-count: 1;       /* or 'infinite' */
  animation-direction: normal;        /* or 'reverse', 'alternate' */
  animation-fill-mode: forwards;      /* keep end state */
}
üí° Performance Tip
Only animate transform and opacity for smooth 60fps animations. Avoid animating width, height, margin, padding.

function Card({ isActive }) {

üìè Method 2: Custom useMediaQuery Hook

  useMediaQuery.js
import { useState, useEffect } from 'react';

function useMediaQuery(query) {
  const [matches, setMatches] = useState(
    () => window.matchMedia(query).matches
  );

  useEffect(() => {
    const mediaQuery = window.matchMedia(query);
    
    const handleChange = (e) => {
      setMatches(e.matches);
    };

    mediaQuery.addEventListener('change', handleChange);
    
    return () => {
      mediaQuery.removeEventListener('change', handleChange);
    };
  }, [query]);

  return matches;
}

export default useMediaQuery;

2.5 Best Practices Summary
React
‚ñº

Making the Right Choice

üéØ Recommended Stack

For Most React Projects:

  • CSS Modules - Component styles (scoped, full CSS)
  • Inline Styles - Dynamic values from props/state
  • Global CSS - Variables, resets, typography
  • clsx - Conditional class management

üìã Decision Guide

Scenario Best Choice
Quick prototyping, utility-first Tailwind CSS
Need Bootstrap components React-Bootstrap
Component library, design system CSS Modules or Styled Components
Dynamic theming, complex states Styled Components
Simple projects, small team CSS Modules

‚úÖ Golden Rules

‚úÖ DO

  • Use CSS variables for theming
  • Design mobile-first
  • Co-locate styles with components
  • Use semantic class names
  • Test on real devices

‚ùå DON'T

  • Mix too many styling methods
  • Use !important
  • Hardcode pixel values everywhere
  • Forget about accessibility
  • Ignore browser support
