What is React?

React.js (or just React) is one of the most popular frontend JavaScript libraries. The creators call it "The library for web and native user interfaces."

üîë Key Concept: What is a Library?

A library is a collection of functionalities that you can use in your code to achieve results that would normally require more code and work. Libraries help you write more concise, less error-prone code and enable you to implement features more quickly.

React is a library that focuses on providing functionalities that help you create interactive and reactive UIs.

The Problem with "Vanilla JavaScript"

Vanilla JavaScript refers to JavaScript without any frameworks or libraries. While it has the advantage of smaller download sizes, the downside is that you must implement all functionalities from scratch.

‚ùå Vanilla JS Example (Imperative)

  .querySelector('p');

  'click',

Step-by-step instructions telling the browser exactly HOW to do things.

‚úÖ React Example (Declarative)

  App.jsx
import { useState } from 'react';

function App() {
  const [outputText, setOutputText] = 
    useState('Initial text');

  function updateTextHandler() {
    setOutputText('Text was changed!');
  }

  return (
    <>
      <button onClick={updateTextHandler}>
        Click to change text
      </button>
      <p>{outputText}</p>
    </>
  );
}

Describe WHAT you want, React handles HOW.

üí° When to Use React
React is NOT a great solution for very simple UIs. If you can solve a problem with a few lines of vanilla JavaScript, there's probably no strong reason to use React. React shines when building complex, interactive UIs.

Single Page Applications (SPAs)

React can be used to manage parts of a website OR the entire page. When React controls all subpages and routing, it's called a Single Page Application (SPA).

üîë What is an SPA?

A website that contains only one HTML file (typically named index.html), which loads the React JavaScript code. The entire UI is then created and managed by JavaScript via React. URL changes are handled by React (client-side routing) without loading new HTML pages from the server.

The Anatomy of a Component

  SubmitButton.jsx
import { useState } from 'react';

function SubmitButton() {
  const [isSubmitted, setIsSubmitted] = useState(false);

  function handleSubmit() {
    setIsSubmitted(true);
  }

  return (
    <button onClick={handleSubmit}>
      {isSubmitted ? 'Loading‚Ä¶' : 'Submit'}
    </button>
  );
}

export default SubmitButton;

Key Observations:

  • Component functions use PascalCase names (e.g., SubmitButton)
  • Inner functions use camelCase (e.g., handleSubmit)
  • Component functions return JSX code (HTML-like syntax)
  • Features like useState() can be used inside components
  • Components are exported to be used elsewhere

What Makes a Function a Component?

A function qualifies as a React component if it returns something that can be rendered by React (typically JSX code). Once it qualifies, it can be used like an HTML element in JSX code.

JSX vs HTML vs JavaScript

JSX is a syntax extension that looks like HTML but is actually syntactical sugar that gets transformed to regular JavaScript.

JSX Code (What you write)

function Greeting() {
  return <p>Hello World!</p>;
}
‚Üí

Transformed Code (What runs)

function Greeting() {
  return React.createElement(
    'p', 
    {}, 
    'Hello World!'
  );
}

JSX Rules

‚ùå Invalid: Multiple Root Elements

function App() {
  return (
    <p>Hello</p>
    <p>World</p>
  ); // Error! Two values
}

‚úÖ Valid: Use Fragment

function App() {
  return (
    <>
      <p>Hello</p>
      <p>World</p>
    </>
  ); // Fragment wraps them
}
üí° React Fragment
Use <>...</> or <React.Fragment>...</React.Fragment> to wrap multiple sibling elements without adding extra DOM nodes.

Props Are Immutable

‚ö†Ô∏è Important: Props Are Read-Only
Props are immutable - you cannot change them inside the component. When a component needs different props, it must "ask" its parent to pass new props. The old props are then discarded.

üìù Chapter 3 Summary

  • Props make components configurable and reusable
  • Props are passed like attributes: <Component prop={value} />
  • Access props via the props object or destructuring
  • The children prop contains content between component tags
  • Use default values with destructuring: { size = 100 }
  • Spread operator {...props} forwards all props
  • Rest property ...props collects remaining props
  • Props are immutable - components cannot modify them

Events in React

React lets you add event handlers to your JSX. Event handlers are functions triggered in response to interactions like clicking, hovering, or form inputs.

Adding Event Handlers

  Three Ways to Add Event Handlers
function Button() {
  // 1. Named function
  function handleClick() {
    alert('You clicked me!');
  }

  return (
    <>
      {/* Method 1: Pass function reference */}
      <button onClick={handleClick}>Click 1</button>
      
      {/* Method 2: Inline arrow function */}
      <button onClick={() => alert('Clicked!')}>
        Click 2
      </button>
      
      {/* Method 3: Inline function expression */}
      <button onClick={function() { alert('Clicked!'); }}>
        Click 3
      </button>
    </>
  );
}
‚ö†Ô∏è Pass Function, Don't Call It!
‚úÖ Correct (passing) ‚ùå Wrong (calling)
<button onClick={handleClick}> <button onClick={handleClick()}>
<button onClick={() => alert('!')}> <button onClick={alert('!')}>

Adding () calls the function immediately during rendering!

Event Propagation

Events "bubble" up the tree. You can stop propagation with e.stopPropagation().

function Button({ onClick, children }) {
  return (
    <button onClick={e => {
      e.stopPropagation();  // Stop bubbling
      onClick();
    }}>
      {children}
    </button>
  );
}

Preventing Default Behavior

function Signup() {
  return (
    <form onSubmit={e => {
      e.preventDefault();  // Stop page reload
      alert('Submitting!');
    }}>
      <input />
      <button>Send</button>
    </form>
  );
}

State: Making UIs Dynamic

State is data that, when changed, triggers React to re-render the component and update the UI.

üîë Why State?

Regular JavaScript variables don't trigger UI updates. React doesn't track them. When you need the UI to change based on data, you must use state.

The useState Hook

  useState Syntax
import { useState } from 'react';

function Counter() {
  // Declare state: [currentValue, setterFunction] = useState(initialValue)
  const [count, setCount] = useState(0);

  function handleClick() {
    setCount(count + 1);  // Update state ‚Üí triggers re-render
  }

  return (
    <>
      <p>Count: {count}</p>
      <button onClick={handleClick}>Increment</button>
    </>
  );
}

Complete Example: Email Validation

  EmailInput.jsx
import { useState } from 'react';

function EmailInput() {
  const [errorMessage, setErrorMessage] = useState('');

  function evaluateEmail(event) {
    const enteredEmail = event.target.value;
    
    if (enteredEmail.trim() === '' || !enteredEmail.includes('@')) {
      setErrorMessage('The entered email address is invalid.');
    } else {
      setErrorMessage('');
    }
  }

  return (
    <div>
      <input
        placeholder="Your email"
        type="email"
        onBlur={evaluateEmail}
      />
      <p style={{ color: 'red' }}>{errorMessage}</p>
    </div>
  );
}

Lifting State Up

When two components need to share state, move it to their closest common parent and pass it down via props.

  Lifting State Example
import { useState } from 'react';

// Parent holds the shared state
function Accordion() {
  const [activeIndex, setActiveIndex] = useState(0);

  return (
    <>
      <Panel
        title="About"
        isActive={activeIndex === 0}
        onShow={() => setActiveIndex(0)}
      >
        About content...
      </Panel>
      <Panel
        title="History"
        isActive={activeIndex === 1}
        onShow={() => setActiveIndex(1)}
      >
        History content...
      </Panel>
    </>
  );
}

// Child receives state via props
function Panel({ title, children, isActive, onShow }) {
  return (
    <section>
      <h3>{title}</h3>
      {isActive ? (
        <p>{children}</p>
      ) : (
        <button onClick={onShow}>Show</button>
      )}
    </section>
  );
}
üí° Single Source of Truth
For each unique piece of state, choose one component to "own" it. Instead of duplicating state between components, lift it up to their common parent and pass it down via props.

JSX Rules Cheat Sheet

‚úÖ Do

  • Use PascalCase for component names
  • Return a single root element
  • Self-close empty elements: <img />
