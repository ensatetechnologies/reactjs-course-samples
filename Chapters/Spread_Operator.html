What is the Spread Operator?
Foundation
‚ñº

The Magic of Three Dots

The spread operator (...) is a powerful ES6 feature that "spreads" or "unpacks" elements from arrays or properties from objects. Think of it as opening a box and laying out all its contents.

üéØ Spread = Unpack Everything!
Original Array (Packed)
[ 1 2 3 ]
‚Üí
After Spread (Unpacked)
... 1 2 3

üìù Basic Syntax

  Spread Operator Basics
// The spread operator is three dots: ...

// With Arrays
const numbers = [1, 2, 3];
console.log(...numbers);  // Output: 1 2 3 (not [1, 2, 3])

// With Objects
const person = { name: 'John', age: 25 };
const copy = { ...person };  // Creates a new object with same properties

// With Strings (yes, it works!)
const word = 'Hello';
console.log([...word]);  // Output: ['H', 'e', 'l', 'l', 'o']

üéØ Three Main Use Cases

üì¶
Copy Arrays
Create new arrays without mutation
üîß
Copy Objects
Clone objects with modifications
üîó
Merge/Combine
Join arrays or objects together
üí° Key Insight
The spread operator creates shallow copies. This means it copies the first level of values, but nested objects/arrays are still referenced (not deep copied).

1. Copy an Array

Creating a Copy (Not a Reference!)
original
[ 1 2 3 ]
‚Üí
copy = [...original]
[ 1 2 3 ]

‚úÖ Different array in memory - changes to one don't affect the other!

  Copy Arrays
const original = [1, 2, 3];

// ‚ùå WRONG: This creates a reference, not a copy
const wrong = original;
wrong.push(4);
console.log(original);  // [1, 2, 3, 4] - Original changed!

// ‚úÖ CORRECT: Spread creates a true copy
const copy = [...original];
copy.push(4);
console.log(original);  // [1, 2, 3] - Original unchanged!
console.log(copy);      // [1, 2, 3, 4]

3. Add Elements (Without Mutation)

Adding to Beginning, Middle, or End
[ 0 , ... 1 2 3 ]
‚Üí Add to START
[ ... 1 2 3 , 4 ]
‚Üí Add to END
  Add Elements Immutably
const numbers = [2, 3, 4];

// Add to beginning
const withFirst = [1, ...numbers];       // [1, 2, 3, 4]

// Add to end
const withLast = [...numbers, 5];        // [2, 3, 4, 5]

// Add to both
const withBoth = [1, ...numbers, 5];    // [1, 2, 3, 4, 5]

// Insert at specific position (using slice)
const arr = [1, 2, 4, 5];
const withMiddle = [
  ...arr.slice(0, 2),   // [1, 2]
  3,                    // new element
  ...arr.slice(2)       // [4, 5]
];
console.log(withMiddle);  // [1, 2, 3, 4, 5]

4. Pass Array as Function Arguments

  Function Arguments
const numbers = [5, 2, 8, 1, 9];

// Math.max expects individual arguments, not an array
// Math.max([5, 2, 8, 1, 9]) ‚ùå doesn't work

// Spread unpacks the array into arguments!
const max = Math.max(...numbers);  // Same as Math.max(5, 2, 8, 1, 9)
console.log(max);  // 9

const min = Math.min(...numbers);
console.log(min);  // 1

// Works with any function
function greet(a, b, c) {
  console.log(`Hello ${a}, ${b}, and ${c}!`);
}
const names = ['Alice', 'Bob', 'Charlie'];
greet(...names);  // Hello Alice, Bob, and Charlie!

2. Update Object Properties (Override)

Updating Properties - Order Matters!
person
{ name: "John" age: 25 }
‚Üí
{...person, age: 26}
{ name: "John" age: 26 }

‚ö†Ô∏è Properties listed AFTER spread override existing ones!

  Update Properties
const person = { name: 'John', age: 25, city: 'Mumbai' };

// Update one property
const older = { ...person, age: 26 };
console.log(older);  // { name: 'John', age: 26, city: 'Mumbai' }

// Update multiple properties
const moved = { ...person, age: 26, city: 'Delhi' };
console.log(moved);  // { name: 'John', age: 26, city: 'Delhi' }

// Add new properties
const withJob = { ...person, job: 'Developer' };
console.log(withJob);  // { name: 'John', age: 25, city: 'Mumbai', job: 'Developer' }

// ‚ö†Ô∏è ORDER MATTERS!
const test1 = { age: 30, ...person };  // age will be 25 (person's age wins)
const test2 = { ...person, age: 30 };  // age will be 30 (30 wins)

4. Remove Properties (with Destructuring)

  Remove Properties
const user = {
  id: 1,
  name: 'John',
  password: 'secret123',  // We want to remove this!
  email: 'john@example.com'
};

// Destructure to separate password, collect rest
const { password, ...safeUser } = user;

console.log(password);  // 'secret123' (extracted)
console.log(safeUser);   // { id: 1, name: 'John', email: 'john@example.com' }

// Remove multiple properties
const { password: _, id: __, ...publicData } = user;
console.log(publicData);  // { name: 'John', email: 'john@example.com' }
‚ö†Ô∏è Shallow Copy Warning
Spread only creates a shallow copy. Nested objects are still referenced!
  Shallow Copy Problem
const original = {
  name: 'John',
  address: {           // Nested object!
    city: 'Mumbai',
    zip: '400001'
  }
};

const copy = { ...original };
copy.address.city = 'Delhi';

console.log(original.address.city);  // 'Delhi' üò± Original changed!

// Solution: Deep copy nested objects too
const deepCopy = {
  ...original,
  address: { ...original.address }  // Spread nested object
};
deepCopy.address.city = 'Delhi';
console.log(original.address.city);  // 'Mumbai' ‚úÖ Original safe!

Spread in React (State Updates)
React
‚ñº

‚öõÔ∏è Spread is ESSENTIAL in React

React requires immutable state updates. You must create new objects/arrays instead of modifying existing ones. Spread is the key tool for this!

Why Spread in React?

‚ùå Mutation (WRONG)

// React won't detect this change!
const updateAge = () => {
  user.age = 26;      // Mutating
  setUser(user);      // Same reference
};
// ‚ùå UI won't update!
VS

‚úÖ Spread (CORRECT)

// Create new object with spread!
const updateAge = () => {
  setUser({
    ...user,          // Copy all
    age: 26           // Override age
  });
};
// ‚úÖ UI updates!

üì¶ Array State Updates

  Array Operations in React
import { useState } from 'react';

function TaskList() {
  const [tasks, setTasks] = useState([
    { id: 1, text: 'Learn React', done: false },
    { id: 2, text: 'Build Projects', done: false }
  ]);

  // ADD - spread existing, add new at end
  const addTask = (text) => {
    const newTask = { id: Date.now(), text, done: false };
    setTasks([...tasks, newTask]);
  };

  // REMOVE - filter creates new array
  const removeTask = (id) => {
    setTasks(tasks.filter(task => task.id !== id));
  };

  // UPDATE - map creates new array with updated item
  const toggleTask = (id) => {
    setTasks(tasks.map(task => 
      task.id === id 
        ? { ...task, done: !task.done }  // Spread + update
        : task
    ));
  };

  // EDIT TEXT
  const editTask = (id, newText) => {
    setTasks(tasks.map(task =>
      task.id === id
        ? { ...task, text: newText }
        : task
    ));
  };

  return /* ... */;
}

import { useState } from 'react';

function UserProfile() {
  const [user, setUser] = useState({
    name: 'John',
    email: 'john@example.com',
    settings: {
      theme: 'light',
      notifications: true
    }
  });

  // Update simple property
  const updateName = (name) => {
    setUser({ ...user, name });
  };

  // Update nested property (need to spread nested too!)
  const toggleTheme = () => {
    setUser({
      ...user,
      settings: {
        ...user.settings,
        theme: user.settings.theme === 'light' ? 'dark' : 'light'
      }
    });
  };

  // Update from form input
  const handleChange = (e) => {
    const { name, value } = e.target;
    setUser({ ...user, [name]: value });  // Dynamic key!
  };

  return (
    <form>
      <input
        name="name"
        value={user.name}
        onChange={handleChange}
      />
      <input
        name="email"
        value={user.email}
        onChange={handleChange}
      />
    </form>
  );
}

üìã Spread Props in JSX

  Spreading Props
// Pass all properties as props
const buttonProps = {
  className: 'btn-primary',
  disabled: false,
  onClick: handleClick
};

// Instead of writing each prop:
<button className={buttonProps.className} disabled={buttonProps.disabled} ...>

// Spread them all at once!
<button {...buttonProps}>Click Me</button>

// Override specific props
<button {...buttonProps} disabled={true}>Click Me</button>

// Common pattern: Forward props to child
function CustomButton({ children, ...props }) {
  return (
    <button className="custom-btn" {...props}>
      {children}
    </button>
  );
}

  TaskManager.jsx - Complete App
import { useState } from 'react';

function TaskManager() {
  // State: array of task objects
  const [tasks, setTasks] = useState([]);
  const [inputValue, setInputValue] = useState('');
  const [filter, setFilter] = useState('all'); // 'all', 'active', 'completed'

  // ‚ûï ADD TASK - Spread existing array, add new task
  const addTask = () => {
    if (!inputValue.trim()) return;
    
    const newTask = {
      id: Date.now(),
      text: inputValue.trim(),
      completed: false,
      createdAt: new Date().toLocaleTimeString()
    };
    
    // ‚ú® SPREAD: Create new array with all existing + new task
    setTasks([...tasks, newTask]);
    setInputValue('');
  };

  // üóëÔ∏è DELETE TASK - Filter creates new array without the task
  const deleteTask = (id) => {
    setTasks(tasks.filter(task => task.id !== id));
  };

  // ‚úÖ TOGGLE COMPLETE - Map + Spread to update specific task
  const toggleTask = (id) => {
    setTasks(tasks.map(task =>
      task.id === id
        ? { ...task, completed: !task.completed }  // ‚ú® SPREAD object
        : task
    ));
  };

  // ‚úèÔ∏è EDIT TASK - Map + Spread to update text
  const editTask = (id, newText) => {
    setTasks(tasks.map(task =>
      task.id === id
        ? { ...task, text: newText }  // ‚ú® SPREAD + override
        : task
    ));
  };

  // üßπ CLEAR COMPLETED - Filter for active only
  const clearCompleted = () => {
    setTasks(tasks.filter(task => !task.completed));
  };

  // üìä Calculate statistics (derived state)
  const stats = {
    total: tasks.length,
    completed: tasks.filter(t => t.completed).length,
    active: tasks.filter(t => !t.completed).length
  };

  // üîç Filter tasks for display
  const filteredTasks = tasks.filter(task => {
    if (filter === 'active') return !task.completed;
    if (filter === 'completed') return task.completed;
    return true;
  });

  return (
    <div className="task-manager">
      <h1>üìù Task Manager</h1>
      
      {/* Input Section */}
      <div className="input-group">
        <input
          type="text"
          value={inputValue}
          onChange={(e) => setInputValue(e.target.value)}
          onKeyPress={(e) => e.key === 'Enter' && addTask()}
          placeholder="What needs to be done?"
        />
        <button onClick={addTask}>Add Task</button>
      </div>

      {/* Filter Buttons */}
      <div className="filters">
        {['all', 'active', 'completed'].map(f => (
          <button
            key={f}
            className={filter === f ? 'active' : ''}
            onClick={() => setFilter(f)}
          >
            {f.charAt(0).toUpperCase() + f.slice(1)}
          </button>
        ))}
      </div>

      {/* Task List */}
      <ul className="task-list">
        {filteredTasks.map(task => (
          <li
            key={task.id}
            className={task.completed ? 'completed' : ''}
          >
            <input
              type="checkbox"
              checked={task.completed}
              onChange={() => toggleTask(task.id)}
            />
            <span>{task.text}</span>
            <span className="time">{task.createdAt}</span>
            <button onClick={() => deleteTask(task.id)}>√ó</button>
          </li>
        ))}
      </ul>

      {/* Statistics */}
      <div className="stats">
        <span>Total: {stats.total}</span>
        <span>Active: {stats.active}</span>
        <span>Completed: {stats.completed}</span>
        {stats.completed > 0 && (
          <button onClick={clearCompleted}>Clear Completed</button>
        )}
      </div>
    </div>
  );
}

export default TaskManager;

Tips, Tricks & Common Mistakes
Important
‚ñº

üí° Master the Spread Operator

‚ùå Common Mistakes

1. Forgetting Nested Spread

// ‚ùå Nested object still referenced
const copy = { ...user };
copy.address.city = 'Delhi';
// Original changed! üò±

// ‚úÖ Spread nested objects too
const copy = {
  ...user,
  address: { ...user.address }
};

2. Wrong Order (Objects)

// ‚ùå Override happens before spread
const result = { age: 30, ...person };
// person.age wins!

// ‚úÖ Spread first, then override
const result = { ...person, age: 30 };
// 30 wins!

3. Spreading null/undefined

// ‚ùå This throws an error!
const data = null;
const copy = { ...data }; // Error!

// ‚úÖ Use fallback
const copy = { ...(data || {}) };
// Or optional chaining
const copy = { ...(data ?? {}) };

4. Thinking It's Deep Copy

// Spread = SHALLOW copy only!
// For deep copy use:

// JSON method (simple objects)
const deep = JSON.parse(
  JSON.stringify(original)
);

// Or structuredClone (modern)
const deep = structuredClone(original);

‚ú® Pro Tips

Conditional Spreading

// Add properties conditionally
const user = {
  name: 'John',
  ...(isAdmin && { role: 'admin' }),
  ...(age && { age })
};

Convert to Array

// String to array
[...'hello'] // ['h','e','l','l','o']

// Set to array (removes duplicates)
[...new Set([1,1,2,3])] // [1,2,3]

// NodeList to array
[...document.querySelectorAll('div')]

üìã Cheat Sheet

Operation Syntax Result
Copy array [...arr] New array with same elements
Add to end [...arr, newItem] New array with item at end
Add to start [newItem, ...arr] New array with item at start
Merge arrays [...arr1, ...arr2] Combined array
Copy object {...obj} New object with same props
Update prop {...obj, key: value} New object with updated prop
Merge objects {...obj1, ...obj2} Combined object (later wins)
