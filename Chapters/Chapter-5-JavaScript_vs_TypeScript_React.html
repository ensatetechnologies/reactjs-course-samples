What is TypeScript?
â–¼

TypeScript = JavaScript + Types

TypeScript is a superset of JavaScript that adds optional static typing. It compiles down to plain JavaScript that runs anywhere JavaScript runs.

ğŸŸ¨ JavaScript

  • Dynamic typing (types checked at runtime)
  • More flexible, less verbose
  • Errors discovered when code runs
  • No compilation step needed
  • Smaller learning curve

ğŸŸ¦ TypeScript

  • Static typing (types checked at compile time)
  • More structured, explicit contracts
  • Errors caught before code runs
  • Requires compilation to JavaScript
  • Better IDE support & autocomplete
ğŸ’¡ Key Insight
TypeScript doesn't change how JavaScript works at runtime. It only adds type checking during development and compilation. All types are erased when compiled to JavaScript!

Sample Component (JavaScript)

  src/components/UserCard.jsx JSX
import { useState } from 'react'

// No type definitions needed
function UserCard({ user, onUpdate }) {
  const [isEditing, setIsEditing] = useState(false)
  const [name, setName] = useState(user.name)

  const handleSave = () => {
    onUpdate({ ...user, name })
    setIsEditing(false)
  }

  return (
    <div className="user-card">
      {isEditing ? (
        <input
          value={name}
          onChange={(e) => setName(e.target.value)}
        />
      ) : (
        <h3>{user.name}</h3>
      )}
      <p>Age: {user.age}</p>
      <button onClick={() => isEditing ? handleSave() : setIsEditing(true)}>
        {isEditing ? 'Save' : 'Edit'}
      </button>
    </div>
  )
}

export default UserCard

}

  TodoApp.tsx TS
// Define Todo type
interface Todo {
  id: number
  text: string
  completed: boolean
}

function TodoApp(): JSX.Element {
  const [todos, setTodos] = useState<Todo[]>([])
  const [input, setInput] = useState<string>('')

  // Return type is void
  const addTodo = (): void => {
    if (!input.trim()) return
    setTodos([...todos, {
      id: Date.now(),
      text: input,
      completed: false
    }])
    setInput('')
  }

  // id must be number
  const toggleTodo = (id: number): void => {
    setTodos(todos.map((todo: Todo) =>
      todo.id === id 
        ? { ...todo, completed: !todo.completed }
        : todo
    ))
  }

  const deleteTodo = (id: number): void => {
    setTodos(todos.filter((t: Todo) => t.id !== id))
  }

  return (/* TSX */)
}

Pros & Cons Comparison

Making the Right Choice

ğŸŸ¨ JavaScript

âœ… Pros

  • Lower learning curve - Just learn JavaScript
  • Faster initial development - No type definitions
  • More flexible - Quick prototyping
  • No compilation step - Direct execution
  • Smaller bundle size - No type overhead
  • Easier for beginners - Less syntax to learn
  • Massive ecosystem - All libraries work

âŒ Cons

  • Runtime errors - Bugs discovered late
  • No autocomplete - Limited IDE support
  • Refactoring nightmare - Easy to break things
  • Poor documentation - Props unclear
  • Team scalability - Hard to maintain
  • Type coercion bugs - "1" + 1 = "11"
  • No compile-time checks - Typos slip through

ğŸŸ¦ TypeScript

âœ… Pros

  • Catch errors early - Before code runs
  • Amazing IDE support - Full autocomplete
  • Self-documenting - Types = documentation
  • Safe refactoring - Compiler catches breaks
  • Better for teams - Clear contracts
  • Fewer runtime bugs - Type safety
  • Future-proof - Industry standard

âŒ Cons

  • Learning curve - New syntax to learn
  • More verbose - Extra type annotations
  • Setup complexity - tsconfig, types
  • Compilation required - Extra build step
  • Some libraries lack types - Need @types
  • Can slow down prototyping
  • Type errors can be cryptic

ğŸ¯ When to Use What?

Choose JavaScript When:

  • Building quick prototypes
  • Small, simple projects
  • Learning React fundamentals
  • Solo projects with limited scope
  • Tight deadlines for MVPs
  • Team is new to programming

Choose TypeScript When:

  • Building production applications
  • Medium to large projects
  • Working with a team
  • Long-term maintainability matters
  • Complex state management
  • API integrations with defined schemas
ğŸ’¡ Industry Trend
Most companies now prefer TypeScript for new React projects. According to surveys, 80%+ of React developers use TypeScript in 2024. Learning TypeScript is a valuable investment for your career!

  Cheat Sheet: JS vs TS Syntax
// ========== VARIABLES ==========
// JavaScript
let name = "John"
// TypeScript
let name: string = "John"

// ========== FUNCTIONS ==========
// JavaScript
const add = (a, b) => a + b
// TypeScript
const add = (a: number, b: number): number => a + b

// ========== ARRAYS ==========
// JavaScript
const nums = [1, 2, 3]
// TypeScript
const nums: number[] = [1, 2, 3]

// ========== OBJECTS ==========
// JavaScript
const user = { name: "John", age: 25 }
// TypeScript
interface User { name: string; age: number }
const user: User = { name: "John", age: 25 }

// ========== REACT COMPONENT ==========
// JavaScript (.jsx)
function Button({ label, onClick }) {
  return <button onClick={onClick}>{label}</button>
}
// TypeScript (.tsx)
interface ButtonProps {
  label: string
  onClick: () => void
}
function Button({ label, onClick }: ButtonProps) {
  return <button onClick={onClick}>{label}</button>
}

// ========== REACT HOOKS ==========
// JavaScript
const [count, setCount] = useState(0)
// TypeScript
const [count, setCount] = useState<number>(0)

// ========== EVENT HANDLERS ==========
// JavaScript
const handleChange = (e) => setValue(e.target.value)
// TypeScript
const handleChange = (e: React.ChangeEvent<HTMLInputElement>) => 
  setValue(e.target.value)
