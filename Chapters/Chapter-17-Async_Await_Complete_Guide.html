Understanding Sync vs Async

In synchronous code, each line waits for the previous one to complete. In asynchronous code, some operations can run in the background while other code continues executing.

üç≥ Real-World Analogy: Making Breakfast

Synchronous (Blocking)

1. Start toasting bread... wait 3 min ‚è≥

2. Toast done! Now boil water... wait 5 min ‚è≥

3. Water boiled! Now make coffee... wait 2 min ‚è≥

Total: 10 minutes üò¥

Asynchronous (Non-Blocking)

1. Start toasting bread üçû

2. Start boiling water ‚òï (while toast is toasting)

3. Prepare coffee grounds (while both are working)

Total: ~5 minutes üöÄ

1Ô∏è‚É£ Callbacks (The Old Way)

Pass a function to be called when the async operation completes.

  Callbacks - Simple Example
// Simulating an API call with setTimeout
function fetchUser(userId, callback) {
  setTimeout(() => {
    const user = { id: userId, name: 'John' };
    callback(user);  // Call the callback with data
  }, 1000);
}

// Using it
fetchUser(1, (user) => {
  console.log('Got user:', user);
});
üò± Callback Hell (Pyramid of Doom)
Nested callbacks become unreadable and hard to maintain!
  Callback Hell - Avoid This!
// ‚ùå CALLBACK HELL - Hard to read and maintain!
fetchUser(1, (user) => {
  fetchPosts(user.id, (posts) => {
    fetchComments(posts[0].id, (comments) => {
      fetchReplies(comments[0].id, (replies) => {
        console.log(replies);
        // üòµ 4 levels deep already!
      });
    });
  });
});

2Ô∏è‚É£ Promises (Better)

A Promise represents a future value. It can be pending, fulfilled, or rejected.

  Promises - Chainable!
// Creating a Promise
function fetchUser(userId) {
  return new Promise((resolve, reject) => {
    setTimeout(() => {
      const user = { id: userId, name: 'John' };
      resolve(user);  // Success!
      // reject(new Error('Failed'));  // Or fail
    }, 1000);
  });
}

// Using Promises - Chainable with .then()
fetchUser(1)
  .then(user => fetchPosts(user.id))
  .then(posts => fetchComments(posts[0].id))
  .then(comments => console.log(comments))
  .catch(error => console.error('Error:', error));

// ‚úÖ Much flatter! No more pyramid!

3Ô∏è‚É£ Async/Await (Modern & Best!)

Syntactic sugar over Promises that makes async code look synchronous!

  Async/Await - Clean & Readable!
// ‚úÖ ASYNC/AWAIT - Reads like synchronous code!
async function loadUserData() {
  const user = await fetchUser(1);
  const posts = await fetchPosts(user.id);
  const comments = await fetchComments(posts[0].id);
  
  console.log(comments);
  return comments;

Understanding Promises
Promises
‚ñº

ü§ù Promises: The Foundation

A Promise is an object representing the eventual completion (or failure) of an asynchronous operation. Understanding Promises is essential because async/await is built on top of them.

üìä Promise States

A Promise Can Be In One of Three States
‚è≥

Pending

Initial state, neither fulfilled nor rejected

‚úÖ

Fulfilled

Operation completed successfully

‚ùå

Rejected

Operation failed

üìù Creating Promises

  Promise Basics
// Creating a Promise
const myPromise = new Promise((resolve, reject) => {
  // Do some async operation
  const success = true;
  
  if (success) {
    resolve('Operation successful!');  // Fulfilled
  } else {
    reject(new Error('Operation failed!'));  // Rejected
  }
});

// Consuming a Promise
myPromise
  .then(result => console.log(result))   // Handle success
  .catch(error => console.error(error))  // Handle error
  .finally(() => console.log('Done!'));   // Always runs

‚ú® Async/Await Syntax

async/await makes asynchronous code look and behave like synchronous code, making it easier to read, write, and debug.

üìù Basic Syntax

  Async/Await Basics
// 1. Mark function as async
async function fetchData() {
  // 2. Use await to pause until Promise resolves
  const response = await fetch('https://api.example.com/data');
  const data = await response.json();
  
  return data;  // Automatically wrapped in a Promise
}

// Arrow function syntax
const fetchData = async () => {
  const response = await fetch('https://api.example.com/data');
  return response.json();
};

// Calling async function
fetchData().then(data => console.log(data));

async

  • Placed before a function declaration
  • Makes function return a Promise
  • Enables use of await inside

await

  • Can only be used inside async function
  • Pauses execution until Promise resolves
  • Returns the resolved value

‚ö° Sequential vs Parallel Execution

  Sequential vs Parallel
// ‚ùå SEQUENTIAL - Slow! Each waits for the previous
async function sequential() {
  const user = await fetchUser(1);      // 1 second
  const posts = await fetchPosts();      // 1 second
  const comments = await fetchComments(); // 1 second
  // Total: 3 seconds! üêå
}

// ‚úÖ PARALLEL - Fast! All start at the same time
async function parallel() {
  const [user, posts, comments] = await Promise.all([
    fetchUser(1),
    fetchPosts(),
    fetchComments()
  ]);
  // Total: ~1 second! üöÄ
}

// ‚úÖ Start parallel, await later
async function parallelAlt() {
  // Start all requests immediately (no await yet)
  const userPromise = fetchUser(1);
  const postsPromise = fetchPosts();
  const commentsPromise = fetchComments();
  
  // Now await all of them
  const user = await userPromise;
  const posts = await postsPromise;
  const comments = await commentsPromise;
}

Sequential: When each request depends on the previous one's result.
Parallel: When requests are independent of each other.

Error Handling with try/catch
Critical
‚ñº

üõ°Ô∏è Handling Errors Gracefully

Async operations can fail (network issues, server errors, etc.). Always handle errors to prevent your app from crashing!

üìù try/catch Syntax

  Basic Error Handling
async function fetchUserData() {
  try {
    // Code that might fail
    const response = await fetch('https://api.example.com/user');
    
    // Check if response is OK (status 200-299)
    if (!response.ok) {
      throw new Error(`HTTP error! status: ${response.status}`);
    }
    
    const data = await response.json();
    return data;
    
  } catch (error) {
    // Handle the error
    console.error('Failed to fetch user:', error.message);
    throw error;  // Re-throw if needed
    
  } finally {
    // Always runs (optional)
    console.log('Fetch attempt completed');
  }
}

üéØ Common Error Patterns

  Error Handling Patterns
// Pattern 1: Return default value on error
async function getUsers() {
  try {
    const response = await fetch('/api/users');
    return await response.json();
  } catch (error) {
    console.error(error);
    return [];  // Return empty array as fallback
  }
}

// Pattern 2: Return error object
async function fetchWithStatus(url) {
  try {
    const response = await fetch(url);
    const data = await response.json();
    return { success: true, data, error: null };
  } catch (error) {
    return { success: false, data: null, error: error.message };
  }
}

// Usage
const result = await fetchWithStatus('/api/users');
if (result.success) {
  console.log(result.data);
} else {
  console.error(result.error);
}

// Pattern 3: Custom error types
class NetworkError extends Error {
  constructor(message, status) {
    super(message);
    this.name = 'NetworkError';
    this.status = status;
  }
}

async function fetchData() {
  try {
    const response = await fetch('/api/data');
    if (!response.ok) {
      throw new NetworkError('Request failed', response.status);
    }
    return await response.json();
  } catch (error) {
    if (error instanceof NetworkError) {
      console.log('Network error:', error.status);
    } else {
      console.log('Unknown error:', error);
    }
  }
}

‚ö†Ô∏è Error Handling with Promise.all

  Promise.all Error Handling
// ‚ùå Promise.all fails fast - if ONE fails, ALL fail
try {
  const results = await Promise.all([
    fetchUser(1),
    fetchUser(-1),  // This fails!
    fetchUser(2)
  ]);
} catch (error) {
  // Catches the first error, other results are lost! üò±
}

// ‚úÖ Promise.allSettled - Get all results regardless of failures
const results = await Promise.allSettled([
  fetchUser(1),
  fetchUser(-1),  // This fails
  fetchUser(2)
]);

results.forEach((result, index) => {
  if (result.status === 'fulfilled') {
    console.log(`User ${index}:`, result.value);
  } else {
    console.error(`User ${index} failed:`, result.reason);
  }
});
üö® Common Mistakes
1. Forgetting try/catch: Unhandled promise rejections can crash your app!
2. Swallowing errors: Catching but not logging or handling properly.
3. Not checking response.ok: Fetch doesn't throw on 404/500 errors!

‚öõÔ∏è Using Async in React

In React, we typically fetch data when a component mounts. This is done using the useEffect hook combined with async/await.

‚ö†Ô∏è Important Rule
useEffect callback cannot be async directly!
You must define an async function inside and call it.

üìù Basic Data Fetching Pattern

  Data Fetching in React
import { useState, useEffect } from 'react';

function UserList() {
  // State for data, loading, and error
  const [users, setUsers] = useState([]);
  const [loading, setLoading] = useState(true);
  const [error, setError] = useState(null);

  useEffect(() => {
    // ‚ùå Can't do this: useEffect(() => async () => {...})
    
    // ‚úÖ Define async function inside
    async function fetchUsers() {
      try {
        setLoading(true);
        const response = await fetch('https://jsonplaceholder.typicode.com/users');
        
        if (!response.ok) {
          throw new Error('Failed to fetch users');
        }
        
        const data = await response.json();
        setUsers(data);
        setError(null);
        
      } catch (err) {
        setError(err.message);
        setUsers([]);
      } finally {
        setLoading(false);
      }
    }

    fetchUsers();  // Call the async function
  }, []);  // Empty array = run once on mount

  // Render based on state
  if (loading) return <div>Loading...</div>;
  if (error) return <div>Error: {error}</div>;

  return (
    <ul>
      {users.map(user => (
        <li key={user.id}>{user.name}</li>
      ))}
    </ul>
  );
}

üé® Custom useFetch Hook

  Reusable Fetch Hook
import { useState, useEffect } from 'react';

// Custom hook for data fetching
function useFetch(url) {
  const [data, setData] = useState(null);
  const [loading, setLoading] = useState(true);
  const [error, setError] = useState(null);

  useEffect(() => {
    let isMounted = true;  // For cleanup

    async function fetchData() {
      try {
        setLoading(true);
        const response = await fetch(url);
        
        if (!response.ok) {
          throw new Error(response.statusText);
        }
        
        const result = await response.json();
        
        // Only update state if component is still mounted
        if (isMounted) {
          setData(result);
          setError(null);
        }
      } catch (err) {
        if (isMounted) {
          setError(err.message);
        }
      } finally {
        if (isMounted) {
          setLoading(false);
        }
      }
    }

    fetchData();

    // Cleanup function
    return () => {
      isMounted = false;
    };
  }, [url]);

  return { data, loading, error };
}

// Usage - So clean!
function UserList() {
  const { data: users, loading, error } = useFetch(
    'https://jsonplaceholder.typicode.com/users'
  );

  if (loading) return <div>Loading...</div>;
  if (error) return <div>Error: {error}</div>;

  return (
    <ul>
      {users?.map(user => <li key={user.id}>{user.name}</li>)}
    </ul>
  );
}

import { useState, useEffect } from 'react';

function UserDirectory() {
  const [users, setUsers] = useState([]);
  const [loading, setLoading] = useState(false);
  const [error, setError] = useState(null);
  const [selectedUser, setSelectedUser] = useState(null);

  // Fetch all users
  const fetchUsers = async () => {
    setLoading(true);
    setError(null);
    
    try {
      const response = await fetch(
        'https://jsonplaceholder.typicode.com/users'
      );
      
      if (!response.ok) {
        throw new Error(`HTTP Error: ${response.status}`);
      }
      
      const data = await response.json();
      setUsers(data);
      
    } catch (err) {
      setError(err.message);
    } finally {
      setLoading(false);
    }
  };

  // Fetch single user details
  const fetchUserDetails = async (userId) => {
    try {
      const response = await fetch(
        `https://jsonplaceholder.typicode.com/users/${userId}`
      );
      const user = await response.json();
      setSelectedUser(user);
    } catch (err) {
      console.error('Failed to fetch user details', err);
    }
  };

  // Fetch user's posts (parallel example)
  const fetchUserWithPosts = async (userId) => {
    try {
      // Parallel fetching!
      const [userRes, postsRes] = await Promise.all([
        fetch(`https://jsonplaceholder.typicode.com/users/${userId}`),
        fetch(`https://jsonplaceholder.typicode.com/users/${userId}/posts`)
      ]);
      
      const user = await userRes.json();
      const posts = await postsRes.json();
      
      setSelectedUser({ ...user, posts });
      
    } catch (err) {
      setError('Failed to fetch user data');
    }
  };

  // Delete user (optimistic update)
  const deleteUser = async (userId) => {
    // Optimistic: Update UI immediately
    const previousUsers = [...users];
    setUsers(users.filter(u => u.id !== userId));
    
    try {
      const response = await fetch(
        `https://jsonplaceholder.typicode.com/users/${userId}`,
        { method: 'DELETE' }
      );
      
      if (!response.ok) throw new Error('Delete failed');
      
    } catch (err) {
      // Rollback on error
      setUsers(previousUsers);
      setError('Failed to delete user');
    }
  };

  return (
    <div className="user-directory">
      <h1>üë• User Directory</h1>
      
      <button onClick={fetchUsers} disabled={loading}>
        {loading ? 'Loading...' : 'Fetch Users'}
      </button>
      
      {error && <div className="error">‚ùå {error}</div>}
      
      {loading && <div className="spinner">Loading...</div>}
      
      <div className="user-grid">
        {users.map(user => (
          <div key={user.id} className="user-card">
            <h3>{user.name}</h3>
            <p>{user.email}</p>
            <button onClick={() => fetchUserWithPosts(user.id)}>
              View Details
            </button>
            <button onClick={() => deleteUser(user.id)}>
              Delete
            </button>
          </div>
        ))}
      </div>
      
      {selectedUser && (
        <div className="user-details">
          <h2>{selectedUser.name}</h2>
          <p>{selectedUser.email}</p>
          {selectedUser.posts && (
            <div>
              <h3>Posts ({selectedUser.posts.length})</h3>
              {selectedUser.posts.slice(0, 3).map(post => (
                <div key={post.id}>{post.title}</div>
              ))}
            </div>
          )}
        </div>
      )}
    </div>
  );
}

‚ú® Async/Await Best Practices

‚úÖ Do's

Always Handle Errors

async function fetchData() {
  try {
    const data = await fetch(url);
    return data.json();
  } catch (error) {
    // Always handle!
    console.error(error);
  }
}

Use Loading States

const [loading, setLoading] = useState(false);

setLoading(true);
try {
  await fetchData();
} finally {
  setLoading(false);
}

Use Promise.all for Parallel

// Run in parallel, not sequence
const [a, b, c] = await Promise.all([
  fetchA(),
  fetchB(),
  fetchC()
]);

Check response.ok

const res = await fetch(url);
if (!res.ok) {
  throw new Error(`${res.status}`);
}

‚ùå Don'ts

Don't Forget await

// ‚ùå Missing await!
const data = fetchData();
// data is a Promise, not the result!

// ‚úÖ Correct
const data = await fetchData();

Don't Use async useEffect Directly

// ‚ùå Wrong!
useEffect(async () => {
  await fetchData();
}, []);

// ‚úÖ Correct
useEffect(() => {
  async function load() {
    await fetchData();
  }
  load();
}, []);

Don't Await in Loops (Sequential)

// ‚ùå Slow! Sequential
for (const id of ids) {
  await fetch(id);
}

// ‚úÖ Fast! Parallel
await Promise.all(
  ids.map(id => fetch(id))
);

Don't Ignore Cleanup

// ‚úÖ Prevent memory leaks
useEffect(() => {
  let cancelled = false;
  
  async function load() {
    const data = await fetch();
    if (!cancelled) setData(data);
  }
  
  return () => { cancelled = true; };
}, []);
